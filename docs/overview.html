Overview

Pro-Only Features

This system can be used freely with Animancer Lite, however it uses some Pro-Only Features which you can try out in the Unity Editor but using them in Runtime Builds requires Animancer Pro:

Feature Used By In Animancer Lite without that feature
Animancer Tools These tools were used to set up the Sprites and create the animations included in this system as explained on the Art page. Unlike other Pro-Only features, these tools aren't usable in Animancer Lite and without them you would need to use the tools built into Unity (which are also explained on the Art page). This only affects the asset set up process and makes no difference in regards to Runtime Builds.
Animancer Events The combat system uses events to activate the hit triggers corresponding to each frame of the attack animations. In Runtime Builds, the attack animations will still play but they will not actually deal damage to enemies or break destructible blocks.
Timeline When the player spawns, they play an Introduction animation made using Unity's Timeline package. In Runtime Builds, the player simply goes straight to their Idle state on startup.
Animation Speed The player character Runs by playing the Walk animation at a faster speed. In Runtime Builds, the animation speed doesn't change. To avoid this, we would need to either have a separate Run animation or make another copy of the Walk animation with the modified speed.

Characters

Character Description
Gino The player character.
Gobbler A monster that walks forward until it reaches a wall then turns around. If an enemy is in front of it, it attacks them. This enemy is on a different Team to the others so it will attack them as well as the player.
Gobbat A monster that flies horizontally in a sine wave pattern and turns around if it hits a wall.
Maw Flower A plant monster that can't move and periodically attacks to either side in a fixed pattern.
Naga
Scarecrow		 Monsters that move and attack like Gobblers, but will also turn around if they would walk off a ledge.
